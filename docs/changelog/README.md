# 版本修复新增内容记录

## 使用说明

本文件夹用于记录每个版本修复的新增内容和变更。

**记录规则：**
- 只有在用户明确说"通过"之后，才会将修复内容记录到 changelog
- 未经用户确认的修复不会记录

## 文件命名规范

`v{版本号}_{日期}.md` - 例如：`v2.9.5_2026-05-08.md`

## 记录格式

```markdown
# v2.9.5 - 修复记录

## 修复日期
2026-05-08

## 新增/修复内容

### 1. 整合包拖拽检测机制
- **问题**: 用户将整合包 ZIP 拖拽到模组页面时，会错误地安装所有模组
- **修复**: 在 `mod_utils.gd` 中添加 bundle.json 检测逻辑
  - 检测 ZIP 解压后根目录或子目录中的 `bundle.json`
  - 如果是整合包，返回错误提示用户到整合包页面导入
- **修改文件**:
  - `utils/mod_utils.gd` - 添加整合包检测
  - `modmanager.gd` - 保留 error_type 字段传递给前端
  - `web/js/mods.js` - 添加 is_bundle_not_mod 错误处理

## 用户确认
- [x] 用户已确认通过
```
