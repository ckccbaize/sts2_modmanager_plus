# 联机补丁功能改进设计

**Date:** 2026-04-07
**Topic:** 联机补丁功能改进

## 1. 需求概述

改进联机补丁（Online Patch / Fix Steam）的管理方式：
1. 联机补丁路径可自定义，仅设置后才可启用联机补丁选项
2. 补丁注入方式调整：原文件保留并添加 `.bak` 后缀
3. 游戏关闭后自动恢复补丁（删除注入的补丁，恢复 `.bak` 文件）
4. 启动管理器和刷新时检测 `.bak` 文件并自动修复

## 2. UI 设计

### 2.1 路径配置区域

在设置界面「联机补丁区域」添加路径配置：
- **LineEdit**: 输入联机补丁目录路径
- **浏览按钮**: 打开文件选择器选择目录
- **自动检测按钮**: 尝试自动检测 `fix_steam` 目录

### 2.2 联机补丁复选框

- 位于启动设置区域的联机补丁复选框
- **路径未设置时**: 复选框禁用（`disabled = true`），显示 tooltip 提示「请先设置联机补丁路径」
- **路径已设置时**: 复选框启用

### 2.3 通知气泡

使用现有的通知系统显示：
- `.bak` 文件修复完成：「已修复 X 个联机补丁文件」
- 补丁注入成功：「联机补丁已注入」
- 补丁恢复成功：「联机补丁已恢复」

## 3. 核心逻辑设计

### 3.1 配置存储

在 `config.cfg` 中新增配置项：
```ini
[paths]
game_path=...
save_path=...
fix_steam_path=...  # 新增：联机补丁目录路径
```

### 3.2 补丁注入流程

1. **检查路径**: 读取 `fix_steam_path` 配置
2. **遍历补丁文件**: 遍历联机补丁目录下的所有文件
3. **重命名原文件**: 如果游戏目录中存在对应文件，将其重命名为 `.bak` 后缀
4. **复制补丁文件**: 将联机补丁文件复制到游戏目录

```gdscript
# 原文件: game_path/file.dll
# 备份: game_path/file.dll.bak
# 注入: fix_steam_path/file.dll -> game_path/file.dll
```

### 3.3 补丁恢复流程

1. **遍历注入的补丁文件**: 对每个曾注入的文件
2. **删除注入的补丁**: 删除游戏目录中的补丁文件
3. **恢复原文件**: 将 `.bak` 文件重命名回原文件名
4. **显示通知**: 提示用户补丁已恢复

### 3.4 .bak 文件检测与修复

在以下时机检测游戏目录中的 `.bak` 文件：
- `_ready()` 启动时
- 刷新按钮点击时

检测逻辑：
1. 扫描游戏目录及其子目录
2. 查找所有 `.bak` 后缀文件
3. 对于每个 `.bak` 文件：
   - 获取原始文件名（去掉 `.bak`）
   - 如果原始文件不存在或被补丁覆盖，则重命名 `.bak` 回原始文件名
   - 删除注入的补丁文件（如果存在）
4. 显示通知告知用户修复结果

### 3.5 补丁恢复时机

使用 `OS.is_process_running(process_id)` 轮询检测游戏进程：
- 启动游戏后，每秒检查一次进程是否结束
- 进程结束后，执行 `_restore_fix_steam_backup()` 恢复补丁
- 显示通知：「联机补丁已恢复，游戏已关闭」

## 4. 文件操作详细设计

### 4.1 注入操作 (`_apply_fix_steam_patch`)

```
fix_steam_path/
├── winmm.dll
├── OnlineFix64.dll
├── OnlineFix.ini
├── dlllist.txt
└── data_sts2_windows_x86_64/
    └── steam_api64.dll
```

对每个文件 `file_name`：
1. 检查游戏目录是否存在 `file_name`
2. 如果存在 → 重命名为 `file_name.bak`
3. 从联机补丁目录复制 `file_name` 到游戏目录

特殊处理 `data_sts2_windows_x86_64` 目录：
- 只复制其中的 `steam_api64.dll`，不覆盖整个目录

### 4.2 恢复操作 (`_restore_fix_steam_backup`)

对每个注入的文件 `file_name`：
1. 检查游戏目录是否存在注入的补丁文件
2. 如果存在 → 删除
3. 检查是否存在 `file_name.bak`
4. 如果存在 → 重命名为 `file_name`

### 4.3 .bak 修复检测 (`_check_and_fix_bak_files`)

扫描游戏目录，查找所有 `*.bak` 文件：
1. 记录所有 `.bak` 文件
2. 对每个 `.bak` 文件：
   - 原始路径 = `.bak` 文件路径（去掉 `.bak` 后缀）
   - 删除原始路径（被补丁覆盖的版本）
   - 将 `.bak` 重命名为原始文件名
3. 返回修复的文件数量

## 5. 数据流

```
config.cfg
├── [paths]
│   ├── game_path
│   ├── save_path
│   └── fix_steam_path  ← 新增
└── [settings]
    └── enable_fix_steam

运行时状态:
- mods[] / enabled_mods{}
- fix_steam_path (从 config 读取)
- enable_fix_steam (从 config 读取)
```

## 6. 错误处理

| 场景 | 处理 |
|------|------|
| 联机补丁路径未设置 | 复选框禁用，显示 tooltip |
| 联机补丁目录不存在 | 注入失败，显示错误通知 |
| 注入时原文件无法重命名 | 跳过该文件，记录日志 |
| 恢复时 .bak 文件不存在 | 跳过该文件，记录日志 |
| 游戏进程检测失败 | 超时后放弃恢复，记录日志 |

## 7. 验收标准

1. **路径配置**: 用户可在设置界面设置联机补丁路径
2. **启用逻辑**: 路径未设置时复选框禁用，路径设置后复选框启用
3. **注入流程**: 注入时原文件重命名为 `.bak`，补丁文件复制到游戏目录
4. **恢复流程**: 游戏关闭后自动删除注入的补丁，恢复 `.bak` 文件
5. **检测修复**: 启动和刷新时自动检测 `.bak` 文件并修复
6. **通知反馈**: 各操作完成时显示通知气泡
